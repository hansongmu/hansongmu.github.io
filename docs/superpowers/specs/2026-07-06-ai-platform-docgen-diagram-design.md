# AI 플랫폼 — 문서 생성 품질 검증 루프 다이어그램 추가

날짜: 2026-07-06 / 승인: 사용자 (설계안 그대로 진행)

## 목적

ai-platform 페이지의 '문서 생성' 섹션은 코드 기반 생성 → 규칙 검사 → 시각 QA → 재생성
루프를 문단 3개로만 설명한다. 기존 흐름도(예: bi/165)와 같은 방식으로, 최종적으로 어떤
워크플로우 형태로 도는지를 보여주는 다이어그램을 추가하고 서술을 한 문장 보완한다.

## 사실 근거 (ai-app 소스 검증)

`ai-app/ai-backend/graph/tools/run_python.py` 기준:

- 에이전트가 docgen/slidegen 코드를 작성 → 격리 샌드박스에서 실행
- 좌표 기반 정적 가드(겹침·영역 이탈·빈 카드) → 결함 시 재시도 유도
- paginated 문서(docx/pptx/pdf)는 페이지 이미지로 렌더 → 멀티모달 모델이 결함 판정
- 결함 발견 시 카드 등록 대신 결함 내역을 ToolMessage로 되돌려 코드 수정·재실행
  (`_MAX_VISUAL_RETRIES = 2`, 초과 시 best-effort 등록)
- 검수는 카드 등록 '전' — 결함본은 사용자에게 노출되지 않음
- templatefill(양식 보존)은 검수 우회 — 다이어그램에는 넣지 않음 (기존 서술 유지)
